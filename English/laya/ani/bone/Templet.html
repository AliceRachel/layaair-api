buildArmature()method
public function buildArmature(aniMode:int = 0):Skeleton

Create animation 0, using template buffered data, template buffered data is not allowed to be modified (memory overhead is small, computation cost is small, and no reloading is required). 1, using animation's own buffers, each animation has its own buffer, which consumes memory (memory cost is large, computation cost is small, and support is changed). 2, use dynamic mode to paint in real time (small memory cost, large computation cost, support for reloading, not recommended use).

Parameters

aniMode:int (default = 0) — 0 animation mode, 0: does not support changing, 1,2 supports dressing

Returns
