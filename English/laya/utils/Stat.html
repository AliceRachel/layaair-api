Stat It is a performance statistics panel that can update relevant performance parameters in real time.

The performance parameters of the statistics are as follows (all parameters are updated every one second or so): The frame rate of FPS (Canvas) /FPS (WebGL):Canvas mode or WebGL mode, that is, the number of frames displayed per second, the higher the value and the more stable, the more fluent the feeling is. Sprite: statistics the number of all rendering nodes (including containers), and its size will affect the efficiency of nodes traversal, data organization and rendering. The smaller the value, the higher the efficiency of the game. DrawCall: this value is an important indicator of performance. The smaller the value, the higher the efficiency of the game. In the Canvas mode, the number of drawing times per second is represented; in the WebGL mode, the rendering of about one second per second is submitted for batch. The process of preparing data and notifying GPU rendering is called the 1 DrawCall. In addition to the time consuming of each DrawCall in the notification GPU rendering, it is also very time-consuming to switch the material and shader. Operation; In CurMem:Canvas mode, the size of memory occupies, the smaller the better the better, too high will result in the game flashing; in the WebGL mode, the memory and memory occupy, the smaller the better the better; Shader: the unique performance index of WebGL mode, which indicates the number of Shader submitted every one second, the smaller the value, the better.
