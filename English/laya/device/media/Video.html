Video Display the video to the Canvas. Video It may not be valid in all browsers.

For all the events supported by Video, see http://www.w3school.com.cn/tags/html_ref_audio_video_dom.asp。

Be careful： At the PC side, it can be called at any time play() Therefore, Video can be played when the program starts running. But at the mobile end, play () can be called only after the user touches the screen for the first time, so it is impossible for the mobile end to automatically start playing Video when the program starts running.

bufferedproperty
buffered:*  [read-only]

The buffered attribute returns to the TimeRanges (JS) object. The TimeRanges object represents the user's audio and video buffer range. The buffer range refers to the time range of buffered audio and video. If the user jumps in the audio and video, it will get multiple buffer ranges.

Buffered.length returns the number of buffers. For example, the first buffer range is buffered.start (0) and buffered.end (0). In seconds.

playbackRateproperty 
playbackRate:Number

The playbackRate property sets or returns the current playback speed of the audio / video. Such as:

  • 1 normal speed
  • 0.5 half speed (more slow)
  • 2 speed (faster)
  • -1.0 back, normal speed
  • -0.5 backward, half speed

Only Google Chrome and Safari support playbackRate properties.

preloadproperty 
preload:String

The preload property sets or returns whether to load the video immediately after the page is loaded. It can be assigned as follows:

  • Auto indicates that once the page is loaded, the video is loaded.
  • Metadata indicates that the audio / video metadata is only loaded when the page is loaded.
  • None indicates that the audio / video should not be loaded after the page is loaded.

readyStateproperty 
readyState:*  [read-only]

Represents the ready state of the video element:

  • 0 = HAVE_NOTHING - no information on audio / video readiness.
  • 1 = HAVE_METADATA - metadata about audio / video readiness
  • 2 = HAVE_CURRENT_DATA - data on the current playback position is available, but there is not enough data to play the next frame / millisecond.
  • 3 = HAVE_FUTURE_DATA - data available at present and at least next frame are available.
  • 4 = HAVE_ENOUGH_DATA - available data is enough to start playing.
