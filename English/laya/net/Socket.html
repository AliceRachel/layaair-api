Socket The HTML5 WebSocket is encapsulated, allowing real-time communication between the server side and the client for full duplex (full-duplex), and allowing cross domain communication. After the connection is established, both the server and Browser/Client Agent can send or receive text and binary data on their own initiative.

To use Socket Class method, please use the constructor first new Socket Create a Socket Object. Socket Transmit and receive data in an asynchronous manner.

protocolsproperty 
public var protocols:*

Subprotocol name. A subprotocol name string or array consisting of multiple sub protocol name strings. It must be assigned before calling connect or connectByUrl, otherwise it will not work.

After specifying, only when the server selects one of the sub protocols, can the connection be established successfully, otherwise the establishment fails and the Event.ERROR event is distributed.

Constructor Detail
Socket()Constructor
public function Socket(host:String = null, port:int = 0, byteClass:Class = null)

Create new Socket Object. The default byte order is Socket.BIG_ENDIAN. If no parameter is specified, a socket that is initially disconnected is created. If you specify a valid parameter, try connecting to the specified host and port.

Be careful： It is strongly recommended to use Without parameters The constructor function, add any event listener and set protocols and other attributes, and then use the host and port parameters to invoke the connect method. This sequence ensures that all event listeners and other related processes are working properly.

Parameters
host:String (default = null) — The server address.
 
port:int (default = 0) — Server port.
 
byteClass:Class (default = null) — The Byte class used to receive and send data. If NULL is used, the Byte class can also be used and the subclass of the Byte class can also be imported.

See also

laya.utils.Byte
