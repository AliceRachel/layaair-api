Classpublic class LoaderManager
InheritanceLoaderManager EventDispatcher Object

LoaderManager Classes are used for batch loading of resources. This is a singleton. Do not manually instantiate this class. Please visit through Laya.loader.

All queues are loaded and the Event.COMPLETE event is dispatches; if any load in the queue fails, the Event.ERROR event is sent, and the event callback parameter is the resource address of the loading error.

LoaderManager The class provides the following functions: Multithreading: the default 5 loading threads can modify the number of threads through the maxLoader attribute. Multiple priority: there are 0-4 priorities in 5, and the priority is high priority loading. 0 is the highest, and the 4 is the lowest. Repetitive filtering: automatic filtering and repeated loading (no simultaneous loading of multiple addresses at the same time) and reuse of cache resources to prevent repeated loading;

clearTextureRes()method 
public function clearTextureRes(url:String):void

Destroy the image resources used by Texture, retain the texture shell, and automatically recover if the image resources used by texture are not available when rendering next time. Compared to clearRes, clearTextureRes just clean up the picture resources used in the texture, do not destroy the texture, and automatically restore the picture resource when it is used again. And clearRes will destroy texture thoroughly, causing no more use; clearTextureRes ensures immediate destruction of picture resources without worrying about destroying errors, and clearRes is destroyed by reference counting. [note] if the picture itself is in the automatic collection (the default picture is less than 51212), the memory can not be destroyed, and this picture is managed by a large collection Manager.

Parameters

url:String — Atlas address or texture address, such as Loader.clearTextureRes ("res/atlas/comp.atlas"); Loader.clearTextureRes ("hall/bg.jpg");

create()method 
public function create(url:*, complete:Handler = null, progress:Handler = null, clas:Class = null, params:Array = null, priority:int = 1, cache:Boolean = true, group:String = null):*

Create an uninitialized resource based on the clas type, then load it asynchronously, when the resource is loaded, the resource of the object is initialized, and the Event.LOADED event is sent through this object, and the parameter value of the event callback is the object itself. The sub resources that embed resources will retain the part after the resource path "."

If URL is an array, return to true; otherwise, return to the specified resource class object, you can judge whether the resource has been loaded by listening to the Event.LOADED event of the object.

Be careful： The cache parameter can only cache the resources of the file suffix atlas. Other resources will ignore the cache and force the reload.

Parameters

url:* — Resource address or array. If URL and clas specify the resource type at the same time, give priority to using the resource type specified by URL. Parameter forms such as: [{url:xx,clas:xx,priority:xx,params:xx},{url:xx,clas:xx,priority:xx,params:xx}] .
 
complete:Handler (default = null) — Load the end callback. According to the URL type, there are 2 cases: 1. URL is a String type, that is, a single resource address, if the load is successful, the callback parameter value is a loaded resource, otherwise null; 2. URL is an array type, and a set of resources to be loaded is specified, if all loads are successfully loaded, the callback parameter value is true, otherwise For false.
 
progress:Handler (default = null) — The resource loads the progress callback, and the callback parameter is the progress information loaded by the current resource (0-1).
 
clas:Class (default = null) — Resource class name. If URL and clas specify the resource type at the same time, give priority to using the resource type specified by URL. Parameter shape, such as: Texture.
 
params:Array (default = null) — Resource construction parameters.
 
priority:int (default = 1) — (default = 1) loading priority, high priority load priority. 0-4 are 5 priorities, 0 are the highest, and 4 are the lowest.
 
cache:Boolean (default = true) — Whether the loaded resources are cached.
 
group:String (default = null)

Returns
* — If URL is an array, it returns true; otherwise, it returns the specified resource class object.

load()method 
public function load(url:*, complete:Handler = null, progress:Handler = null, type:String = null, priority:int = 1, cache:Boolean = true, group:String = null, ignoreCache:Boolean = false):LoaderManager

Load the resources. When the resource is loaded wrong, the object will distribute the Event.ERROR event, and the callback parameter value of the event is the wrong resource address.

Because the return value is LoaderManager The object itself, so you can use the following syntax: Laya.loader.load (...).Load (...);

Parameters

url:* — An array of individual resource addresses or resource information to load. For example: simple array: ["a.png","b.png"] Complex array [{url:"a.png",type:Loader.IMAGE,size:100,priority:1},{url:"b.json",type:Loader.JSON,size:50,priority:1}] .
 
complete:Handler (default = null) — Load the end callback. According to the URL type, there are 2 cases: 1. URL is a String type, that is, a single resource address, if the load is successful, the callback parameter value is a loaded resource, otherwise null; 2. URL is an array type, and a set of resources to be loaded is specified, if all loads are successfully loaded, the callback parameter value is true, otherwise For false.
 
progress:Handler (default = null) — Load progress callback. The callback parameter is the loading progress information of the current resource (0-1).
 
type:String (default = null) — Resource types. For example: Loader.IMAGE.
 
priority:int (default = 1) — (default = 1) loading priority, high priority load priority. 0-4 are 5 priorities, 0 are the highest, and 4 are the lowest.
 
cache:Boolean (default = true) — Whether to cache the result of the load.
 
group:String (default = null) — It is convenient to manage the resources by grouping.
 
ignoreCache:Boolean (default = false) — Whether to ignore caching and force to reload.

Returns
LoaderManager — this LoaderManager The object itself.
