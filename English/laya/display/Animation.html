Animation It is the Graphics animation class. A Graphics based animation creation, playback and control interface is implemented.

This class uses an animated template cache pool, which saves CPU overhead with a certain memory overhead. When the same animation template is used many times, compared to creating a new animation template each time, using the animated template cache pool, it only needs to be created once, multiple reuse after the cache, thus saving the overhead of animation template creation.

Animation template cache pool, with key-value key value to storage, key can be custom, can also be read from the specified configuration file, value as the corresponding animation template, is an array of Graphics objects, each Graphics object corresponds to a frame image, the essence of animation is to switch Graphics objects on time.

You can use set source, loadImages (...), loadAtlas (...) and loadAnimation (...) to create animation templates. The specified animation can be played with play (...).

loadAnimation()method 
public function loadAnimation(url:String, loaded:Handler = null, atlas:String = null):Animation

Load and analyze animation files produced by LayaAir IDE, which may contain multiple animations. The default frame rate is the frame rate designed in IDE. If set interval is called, the frame rate corresponding to this frame interval is used. After loading, the animation template is created and cached to the animation template cache pool, key "url# animation name" corresponds to the animation template of the corresponding animation name, and the key "url#" corresponds to the default animation template of the animated template collection.

Note: if you do not preload the atlas used by animation before calling this method, please specify the atlas parameter as the corresponding atlas path, otherwise the animation creation will fail.

The animation template caching pool saves CPU overhead with a certain memory overhead. When the same animation template is used many times, compared to creating a new animation template each time, using the animation template cache pool, it only needs to be created once, multiple reuse after the cache, thus saving the overhead of the creation of the animation template.

Because the return value is Animation object itself, you can use the following syntax: ani.loadAnimation (...).LoadAnimation (...).Play (...);

Parameters

url:String — Animated file path. It can be created and published by LayaAir IDE.
 
loaded:Handler (default = null) — (optional) use the specified animation resource to initialize the callback completed by animation.
 
atlas:String (default = null) — (optional) the atlas address used by the animation (optional).

Returns
Animation — Return to the animation itself.
loadAtlas()method 
public function loadAtlas(url:String, loaded:Handler = null, cacheName:String):Animation

Initializes the current animation sequence frame according to the specified animation template. The process of selecting the animated template is as follows: 1. animated template key for cacheName in the animation template cache; 2. if no, the specified atlas is loaded and the animation template is created.

Note: only by specifying the cacheName that is not empty can the created animation template be cached for key to the animation template buffer pool, otherwise it will not be cached.

The animation template caching pool saves CPU overhead with a certain memory overhead. When the same animation template is used many times, compared to creating a new animation template each time, using the animation template cache pool, it only needs to be created once, multiple reuse after the cache, thus saving the overhead of the creation of the animation template.

Because the return value is Animation object itself, you can use the following syntax: ani.loadAtlas (...).LoadAtlas (...).Play (...);

Parameters

url:String — The atlas path. When you want to create an animation template, you will use this as the data source.
 
loaded:Handler (default = null) — (optional) initialize the callback finished with the specified atlas.
 
cacheName:String — (optional) key of animation template in animation template buffer pool. If this parameter is not empty, it means using the animation template buffer pool. If the animation template exists in the animation template buffer pool, use this template as key's cacheName animation template. Otherwise, create a new animation template, if cacheName is not empty, cacheName is cached in the animated template cache pool with cacheName key, and if cacheName is empty, no caching is done.

play()method 
override public function play(start:* = 0, loop:Boolean = true, name:String, showWarn:Boolean = true):void

Start playing animation. An animated template with a key value of name is found in the animated template cache pool, and the current sequence frame is initialized with this animated template, and the current sequence frame is used if it does not exist.

The play (...) method is designed to be called at any time after creating an instance, after the call is in the playback state, and when the corresponding resource is loaded, the animated frame filling method (set frames) is called or the instance is displayed on the stage, it will judge whether it is in the playback state, and if so, it will start playing.

With the wrapMode attribute, the animation play sequence type can be set.

Parameters

start:* (default = 0) — (optional) specify the index (int) or the String of the start of the animation. The frame labels can be added and deleted by addLabel (...) and removeLabel (...).
 
loop:Boolean (default = true) — (optional) whether to circulate.
 
name:String — (optional) the key of the animation template in the animation template buffer pool, or the animation name. If name is empty, the current animation sequence frame is played; if not empty, an animated template with a key value of name is found in the animated template cache pool, if there is an animated template that initializes the current sequence frame and plays, if not, the current animated sequence frame is still played; if there is no frame data of the current animation It does not play, but the instance is still playing.
 
showWarn:Boolean (default = true)
