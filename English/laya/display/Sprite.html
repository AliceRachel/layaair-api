Sprite A display list node that is the basic display of the graphics. Sprite The default is no width height, and the default is not to accept mouse events. adopt graphics You can draw pictures or vector diagrams to support rotation, zooming, displacement and other operations. Sprite It is also a container class that can be used to add multiple child nodes.

Be careful： Sprite The default is not wide and can pass through getBounds Function can be obtained; manual width width can also be set. autoSize=true And then get the width. Sprite The width is generally used for collision detection and typesetting, and does not affect the size of the display image. If you need to change the size of the display image, please use it. scaleX ， scaleY ， scale .

Sprite Default does not accept mouse events, that is mouseEnabled=false However, as long as any mouse event is monitored, it will automatically open itself and all parent objects. mouseEnabled=true . So it doesn't normally need to be manually set mouseEnabled .

The design of the LayaAir engine API is simple and ingenious. The core display class has only one Sprite . Sprite Rendering optimization is done for different situations, so that a class can achieve rich functions while achieving high performance.

cacheAsproperty 
cacheAs:String

Specifies whether the display object is cached as a static image, when the cacheAs, the subobject changes, will automatically recache, and can also manually call the reCache method to update the cache.

It is recommended to cache complex content that is not changing frequently into static images, which will greatly improve rendering performance. CacheAs has three values of "None", "normal" and "bitmap".

  • The default is "None" and does not do any caching.
  • When the value is "normal", the canvas mode will be used to perform the cache of the canvas and the command cache under the webgl mode.
  • When the value is "bitmap", the canvas mode is still the canvas cache, and the renderTarget cache is used in the webgl mode.

    • Webgl renderTarget cache mode shortcoming: additional renderTarget objects will be created to increase memory overhead, the cache area has the maximum 2048 limits, and the CPU overhead will increase when redrawing. Advantages: reduce drawcall dramatically and render the highest performance.

    hitTestPriorproperty 
    public var hitTestPrior:Boolean = false

    Specifies whether mouse event detection is prior to detecting itself or giving priority to detecting its sub objects. Mouse event detection occurs during the capture phase of the mouse event, in which the engine recursively detects stage and its subobjects until the target object is found or no object is hit by the stage.

    If false is used, the sub object is detected first. When a sub object is hit, the interrupt is detected and the hit target is obtained. If it is not hit any subobject, then the object is then detected; if it is true, the object is detected first, if the object is not hit, the detection is interrupted directly, indicating that the target is not hit; if the object is hit, then the subobject is further recursively detected to confirm the final hit target.

    Using this property reasonably can reduce the node of mouse event detection and improve the performance. It can be set to the case of true: the developer does not care about the mouse event detection results of the child node of this node, that is, the node as the mouse event detection basis for its sub nodes.

    The Stage object and the View component of the UI default to true.

    The default value is false 优先检测此对象的子对象，当递归检测完所有子对象后，仍然没有找到目标对象，最后再检测此对象。.

    mouseThroughproperty 
    public var mouseThrough:Boolean = false

    Whether the collision detection between the mouse event and this object is penetrable. Collision detection occurs during the capture phase of the mouse event, in which the engine recursively detects stage and its subobjects until the target object is found or no object is hit by the stage.

    When the penetration indicates that the location of the mouse event is in the area of the object drawing area, the hit is counted, and the hitArea attribute of the Rectangle object is independent of the height of the object and the value of the object. If the sprite.hitArea value is a HitArea object, it means that the mouse event response area of the object is explicitly declared, ignoring the width and mouseThrough attributes of the object.

    The attribute of the mouse event response area of the object is width, height, and hitArea, and the priority order is: hitArea (type:HitArea) >hitArea (type:Rectangle) >width/height.

    The default value is false 不可穿透，此对象的鼠标响应区域由width、height、hitArea属性决定。
